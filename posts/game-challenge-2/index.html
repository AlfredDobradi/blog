Game Challenge(?) - continued

2018-09-12 11:06
Tags: news, game, development

I’ve been thinking about how to actually do the game challenge in term of time restrictions and feasibility and my best idea so far is to have a 36 hour window in which I start with 12 hours of programming (and streaming), then do something else for 12 hours (probably sleeping lol) and finish with another 12 hour session of writing the game.

This would be good because: * I can take a break and sleep * I won’t have enough time to get ideas which result in rewriting the whole thing * It can still be handled as one event

However there are a few things I’ve still to decide. I’m more or less set on what kind of game I want to make but I’m not sure what to use. Here’s the dilemma: I can code confidently in a few languages and since there won’t be any complicated graphics involved, I could just use those but only a handful of them have support for GTK+ (the framework I’ll be using to render the application cross-platform), and some of those aren’t even official so this limits the possibilities. Since this is a challenge, I might as well look into trying something new but that would mean I might not be aware of some common and/or best practices which I’d rather like to avoid. At this point the main contenders are Python and C++ both because they have official bindings for GTK+.

I know a bit of Python but I only have a surface-level knowledge on C++, definitely have to read up on both to be able to create something good in them. Currently Python is probably more favorable but I’ve been looking into learning C++ for quite a while so this could be a great opportunity. It’s always hard to make this decision because nobody likes to scrap a project to switch languages and I also don’t want to waste time during the challenge looking up basic stuff.
