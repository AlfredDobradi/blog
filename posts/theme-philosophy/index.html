Theme Philosophy: DMDM

2018-08-29 10:35
Tags: theme

The theme I’m using for this blog is very barebones. I’m sure there are existing themes like this for Hugo but when the thought of this blog formed in my head I had a few principles regarding how I want to create and maintain it. The first - and most important - point was it all has to be done by me - well all except Hugo - which includes the theme as well.

Even though it’s not a complicated theme at all, when you create something it helps to develop a certain philosophy first and go with that in mind. As you can guess my main goal was creating a very simplistic theme using as few CSS rules as possible:

  • I use Nicholas Gallagher’s Normalize CSS as a boilerplate
  • Control spacing (using margin, padding and width)
  • Control font-family and font-size

It’s not a long list and the code is very minimal, come to think of it, cutting it up to 5 different scss imports was absolutely unnecessary, but I like organizing my code so whatever.

I chose Open Sans for the font family of the theme as it’s a clean and very readable typeface, although I’m not sure if I should use a serif font instead to improve readability further, which is the long-term goal. Minimalistic shouldn’t be unreadable, right? I’ll probably introduce better line and letter spacing over time, but I’ve yet to find a good combination of the two. As for the sizing of the text, the reason behind changing browser defaults is just having control over it and giving it a nice big letter feel. I use 16px size in my editors and I usually prefer big line heights and large text as it’s easier on the eye.

The spacing bit doesn’t really require much explanation I assume, it’s easier to visually separate components and I really like indents for some weird reason.

Long story short, I wanted a simple, minimalistic but readable theme and I wanted to do it myself, so DMDM was born. If you want to get it - to use or to build on - just head over to https://vcs.brvy.win/barveyhirdman/dmdm and give it a try.
